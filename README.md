Server Log Processor (Variant 9)Консольний Java-додаток, призначений для автоматизованого парсингу, фільтрації, валідації та трансформації сирих рядків серверного логу. Проєкт демонструє ефективну роботу з класами String, StringBuilder, а також обробку тексту за допомогою регулярних виразів (Regex).🛠️ Функціонал та етапи обробки логівКлас LogProcessor реалізує конвеєр обробки даних (Варіант 9), що складається з таких кроків:  Очищення та нормалізація: Видалення випадкових початкових/кінцевих пробілів та заміна множинних внутрішніх пробілів на один.  Фільтрація (Виключення за IP): Перевірка рядка за допомогою регулярного виразу ^192\.168\..*. Якщо IP-адреса сервера починається на 192.168., такий лог повністю ігнорується та видаляється з фінального звіту.  Парсинг та валідація: Перевірка структури рядка на відповідність стандартному формату логу за допомогою груп захоплення (Capture Groups) в Regex. Виокремлюються: IP, дата/час, HTTP-метод та URL-шлях. Битi або некоректні рядки відсіюються з виведенням попередження в System.err.  Трансформація даних: Модифікація отриманого HTTP-методу шляхом його реверсу (перевороту тексту задом наперед, наприклад: GET $\rightarrow$ TEG, POST $\rightarrow$ TSOP) за допомогою StringBuilder.  Генерація звіту: Зведення всіх успішно оброблених записів у фінальну текстову таблицю-звіт із вирівнюванням колонок.  🗂️ Структура проєктуПроєкт складається з таких компонентів:ServerLogRecord — незмінна (Immutable) модель даних, що представляє собою одну валідовану та оброблену одиницю логу (IP, Timestamp, реверснутий метод, шлях).  LogProcessor — ядро системи, що містить логіку фільтрації, регулярні вирази для парсингу та метод для побудови текстової таблиці-звіту.  Main — точка входу, що містить набір тестових логів (Mock-даних), запускає процес обробки та виводить фінальний результат у консоль.
